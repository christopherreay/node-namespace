# Rung 8 — scope bridge (experimental)

The scope bridge is the one group in the namespace API that fundamentally cannot be an ordinary function: generating code that creates local variables requires `eval` in the caller's scope. This rung demonstrates the pattern in isolation. The `toEval_bindMustExist` call produces a string of `var` declarations; `eval`'ing it in THIS function's scope binds the values as local variables accessible to all subsequent code in the same function — including closures below. The value is narrow but real: deep tool scenarios where you want to assert a subtree exists AND bind its leaves to local variables in one move, without destructuring boilerplate. Use only where the scope-binding is genuinely the point; prefer `destructureMustExist` (rung 7) for ordinary batch reads.
